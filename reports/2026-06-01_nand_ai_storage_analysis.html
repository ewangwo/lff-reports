NAND不只是SSD：AI正在把存储推向数据中心核心

NAND FlashAI InfraData-CentricCXL3D NANDComputational Storage

AI真正的瓶颈，已经不只是GPU算力，而是数据如何更快、更低功耗地流动起来。

AI推动的不只是GPU革命，而是一场从"计算中心"走向"数据中心"的底层架构重构。

产业趋势转变：过去NAND竞争=容量更大 → 未来NAND竞争=存储架构、数据流、计算协同与AI系统优化

核心内容

1. 全球数据爆炸→NAND超长期扩展周期

云端/边缘/终端数据同时爆发。"NAND is not replaceable."——已成为数据中心不可替代的核心介质。

2. 3D NAND：用垂直堆叠延续摩尔定律

"2D NAND was Litho Driven; 3D NAND is Etch Driven."

层数演进：112层→162层→2xx/4xx/5xx层

四大扩展方向：Vertical Scaling / Lateral Scaling / Architecture Scaling / Logical Scaling

3. 三角挑战：Performance vs Reliability vs Cost

QLC/PLC虽降低Cost/GB，但带来：

  • ⚠ 缩小Vt Window
  • ⚠ 降低耐久度
  • ⚠ 增加误码率
  • ⚠ 恶化Retention

→ 引入LDPC纠错、RAID保护、Wear Leveling、FTL、复杂SSD Controller

→ SSD已经越来越像"一个小型分布式计算系统"

4. SSD：从存储设备→数据管理系统

传统问题：Host无法控制Flash内部行为（GC/Wear Leveling→延迟波动、QoS恶化）

→ Open Channel SSD / ZNS，让Host直接参与数据管理

→ SSD控制权正在从Device转向System Software

5. AI推动"计算靠近存储"——数据中心架构时代

传统（Compute-Centric）瓶颈：IO跟不上、数据移动耗电巨大、CPU/GPU等待数据

未来（Data-Centric）："把计算搬到数据旁边"

关键方向：

  1. Computational Storage — SSD内置FPGA/ASIC，Storage侧AI预处理
  2. DPU — 数据处理单元，减轻CPU负担
  3. Memory Tiering / CXL Memory — CXL连接NAND大容量空间
  4. Compute-in-Memory (CIM) — 存内计算，AI时代重要方向

关键风险

  • NAND周期属性极强，目前上行周期可能反转
  • QLC/PLC可靠性争议可能延缓企业级大规模采用
  • CXL/Computational Storage生态成熟度还不够
