以上两个概念是当今大模型得以高性能运行的关键：Continuous Batching + Paged Attention

核心矛盾与解决

LLM生成式任务的核心矛盾：每个请求的输出序列长度不可预测且差异巨大。

  • 传统批处理：请求步调一致→效率低下
  • Continuous Batching：调度从request level下沉到token level
  • Paged Attention：KV Cache按token level按需申请，block table维护地址

Continuous Batching（连续批处理）

调度器视角中不存在Prefill和Decode的区分。

调度目标：让 num_computed_tokens 追上 num_tokens

4个硬性约束：

  1. 最大并发请求数
  2. Token Budget（控制GPU计算量上限）
  3. 模型最大序列长度
  4. 是否有空闲KV Cache blocks

Paged Attention（分页注意力）

# KV Cache shape: [num_layers, 2, num_blocks, block_size, num_kv_heads, head_dim]
# block_table: [max_num_reqs, max_num_blocks_per_req]
# slot_mapping: 告诉kernel写到哪个slot

slot_idx = slot_mapping[token_idx]
block_idx = slot_idx / block_size
block_offset = slot_idx % block_size

关键设计：block_table不需要num_layers维度——因为一个token在所有层的物理块位置相同。

Trade-off分析

PagedAttention间接寻址引入离散访存（Uncoalesced Access），降低L2 Cache命中率。

→ block_size=16缓解：一个block内16个token物理连续

→ 用少部分带宽折损换取系统吞吐量大幅跃升

LLM推理全流程（Tensor维度变化）

1. Tokenize

BPE（Byte Pair Encoding）→ Token ID数组

2. Embedding Lookup

阶段输入权重输出备注
Embedding[num_sched_tokens][V, hidden_size][num_sched_tokens, hidden_size]Flattened布局，消除Padding

3. RMSNorm

砍掉平移操作，仅沿特征维度做尺度缩放。

输入/输出: [num_sched_tokens, hidden_size] | 权重: [hidden_size]（一维向量，逐通道缩放）

4. Fused QKV Projection

单次宽矩阵GEMM。Q 4096 + K 1024 + V 1024 = 6144。

阶段输入权重输出
Fused QKV[num_sched_tokens, hidden_size][hidden_size, 6144][num_sched_tokens, 6144]

5. RoPE（旋转位置编码）

用旋转角度代表token位置。cos_sin_cache预计算，避免实时sin/cos（SFU吞吐仅为FP32 ALU的1/4）。

→ 空间换时间/访存换算力

6. FlashAttention

关键切换：从Flattened [num_sched_tokens] → 请求维度 [batch_size]

通过 cu_seqlens 数组实现变长序列的请求级隔离。

Attention是LLM推理最复杂环节：数学逻辑不复杂，但为了打破内存墙做了大量优化。
